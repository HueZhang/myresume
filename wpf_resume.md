## 项目经历

### 核心项目

#### 多租户 ERP 桌面客户端（WPF + MVVM / .NET 8 / ASP.NET Core Web API）

> WPF 桌面版 SaaS ERP 系统，MVVM 架构，覆盖零售、采购、仓储、财务、生产制造全模块。通过 Web API 与后端协作，客户端负责交互、渲染与硬件集成，后端提供业务逻辑与数据服务。

- **MVVM 架构设计**：CommunityToolkit.Mvvm 实现 ViewModel ↔ View 双向数据绑定；Messenger 实现跨模块解耦通信（如订单审核后自动触发库存更新通知）；通过 IOC 容器注入服务与仓储，使 ViewModel 层更便于单元测试与逻辑复用
- **XAML 样式与控件**：全局统一资源字典定义主题（配色、字体、间距），DataGrid 通过 CellStyle 绑定实现库存预警标红、状态流转高亮等条件样式；DataTrigger 驱动单据编辑/查看双态切换
- **自定义控件**：封装通用 DataGrid 组件（分页、排序、列冻结、单元格编辑、行拖拽）；自定义下拉选择器支持虚拟化滚动，优化大数据量列表加载与滚动体验；表单控件集成 ValidationRule 客户端校验 + FluentValidation 服务端校验双保险
- **实时数据推送**：SignalR 推送库存异动、订单状态变更至桌面端底栏实时滚动通知；本地 SQLite 缓存离线数据，网络恢复后自动增量同步
- **硬件设备集成**：封装串口通信抽象层（RS232），适配电子秤、条码枪、小票打印机、RFID 读写器等外设；条码枪输入通过全局键盘钩子捕获，自动路由到当前焦点单据行，无侵入式集成
- **打印引擎**：基于 XAML FixedDocument 实现小票、A4 单据、标签等多规格打印，打印模板支持在线定制（页眉/页脚/列配置/水印）
- **审批流**：流程节点可视化配置，审批人按部门/角色/指定人三级可配，最多五级流转，操作历史时间线展示

#### IOT 医疗设备数据采集系统（WinForms + 串口通信 + Vue3 大屏）

> 桌面版医疗设备数据采集终端，通过串口协议对接血压计、体重秤等设备，实时采集、解析、上传测量数据至中心系统，配套数据大屏与叫号大屏。

- **设备对接与协议解析**：基于 RS232/RS485 串口通信，使用 Nmodbus 等协议对接多家厂商的血压计、体重秤等医疗设备，按厂商协议文档实现数据帧解析（校验位、数据位、停止位配置）
- **抽象设备配置方案**：设计设备驱动抽象层，不同厂商设备只需实现统一接口即可接入，新增设备类型时减少对核心采集逻辑的改动，提升系统兼容性与扩展性
- **TCP Socket 在线监测**：TCP 长连接实时监控设备在线状态，离线/异常自动告警，保障临床数据采集不中断
- **数据大屏与叫号大屏**：CEF 内嵌 Vue3 前端替代传统 WinForms 图表控件，实现数据可视化大屏与候诊叫号大屏，支持动态刷新与多分辨率适配
- **服务协作**：调用 API 网关代理接口，间接访问主系统数据并回传测量结果，实现终端与中心系统的松耦合集成

#### 多租户电子发票网关（.NET 8 + MongoDB + 工厂模式）

> 聚合百望、金蝶等多家第三方税务厂商，为多地区提供统一开票服务

- **核心技术**：工厂模式 / MongoDB TTL 缓存 / .NET Framework 4 双框架支持 / Redis 分布式锁 / SM4 加密
- **可扩展设计**：工厂模式标准化百望、金蝶等异构接口，新增开票渠道主要通过注入实现类完成，减少对核心开票流程的影响
- **缓存降本**：MongoDB 保存第三方请求/响应，TTL 自动清理临时数据，减少重复查询与重复调用
- **并发安全**：Redis 分布式锁串行化多渠道开票请求；Redis SETNX 业务流水号幂等校验，防止重复开票
