## 项目经历

### 核心项目

#### 多租户ERP系统（.NET 8/ EFCore / MySQL / Hangfire ）

> 基于 .NET 8的SaaS多租户ERP系统，微服务架构，专注进销存+财务+生产功能。主要模块有零售管理、采购管理、销售管理、仓库管理、财务管理、报表查询、系统管理等。支持预付款、收入支出、仓库调拨、组装拆卸、订单等特色功能。

- **订单状态流转**：未审核 → 审核 → 部分入库/出库 → 完成，支持单据间自动转换。
- **库存多维度管理**：库存按 SKU + 库位 + 货主 + 序列号 + 效期 + 批号多维度管理，支持冻结/解冻、移库、盘点调整等库内操作，使用乐观并发控制保障高并发场景下库存数据一致性。
- **可配置多级通用审批流**：最多支持五级审核，每级可以选择当前部门负责人、上级部门负责人、指定某个人，可以给各类单据配置单独的流程。
- **BOM驱动的生产管理**：计算物料需求 (MRP)；成本核算，系统自动累加 BOM 中所有物料成本+ 制造费用，计算出成品的标准成本。
- **电商平台多端对接**：通过Web API + Adapter适配层架构，统一对接淘宝/天猫、京东、拼多多等主流平台Open API，实现订单自动下载、库存实时同步、物流回传。设计多平台字段映射引擎解决各平台SKU编码差异，增量同步采用时间窗口前移策略防漏单，大促期间QPS动态限流保障API调用稳定性
- **多租户与细粒度权限**：基于 JWT 的认证方案（含 Refresh Token），支持菜单级 + 操作级权限控制，实体层自动过滤租户数据，支持 SaaS 多租户部署。
- **对接硬件设备**：通过串口，键盘模拟等协议，对接电子秤、扫码枪、小票机、条码打印机等物联设备

#### 专家培训与考试系统（.NET 8+ Redis + FFmpeg + WebSocket）

> 支持考试高峰期集中登录、视频学习和交卷提交，通过 Redis 限流、异步转码和幂等提交降低核心接口压力。

- **核心技术**：Redis 限流 / 七牛云分片上传 / FFmpeg 异步转码 / WebSocket 实时推送 / Workflow Core 审批流
- **限流防刷**：Redis 滑动窗口限流，按手机号+IP+设备指纹多维度分桶，缓解考试高峰期集中登录与恶意刷接口压力
- **视频转码**：上传至七牛云 OSS，Webhook 回调触发 Redis 队列，FFmpeg 异步转码为多分辨率（720p/1080p），主流程不阻塞，减少用户等待时间
- **弱网上传**：七牛云分片直传 + 断点续传，优化大文件在弱网环境下的上传稳定性
- **跨节点实时通信**：Redis Pub/Sub 构建跨节点消息路由总线，用户无论连接哪个节点都能收到 WebSocket 实时状态推送
- **接口幂等**：考试交卷采用 Redis SETNX 检查考生（专家） ID+试卷 ID 唯一标识，防止重复提交
- **分布式 ID**：借鉴**雪花算法**思想，基于时间戳、节点标识和随机因子生成考试订单号，适配多节点部署场景下的订单号生成需求。
- **通用审批流**： 支持按部门/角色/身份配置审批人，层级自动流转

#### 多租户电子发票网关（.NET 8 + MongoDB + 工厂模式）

> 聚合百望、金蝶等多家第三方税务厂商，为多地区提供统一开票服务

- **核心技术**：工厂模式 / MongoDB TTL 缓存 / .NET Framework 4 双框架支持 / Redis 分布式锁 / SM4 加密
- **可扩展设计**：工厂模式标准化百望、金蝶等异构接口，新增开票渠道主要通过注入实现类完成，减少对核心开票流程的影响
- **缓存降本**：MongoDB 用于保存第三方请求/响应，解析关键数据保存和短期查询缓存，减少重复查询与重复调用；TTL 自动清理临时数据；
- **并发安全**：Redis 分布式锁串行化多渠道开票请求；Redis SETNX 业务流水号幂等校验，防止重复开票
