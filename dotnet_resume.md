## 项目经历

### 核心项目

#### 专家培训与考试系统（.NET 10 + Redis + WebSocket）

> 高并发在线考试平台，支撑峰值多人同时登录与考试

- **核心技术**：Redis 限流 / 七牛云分片上传 / FFmpeg 异步转码 / WebSocket 实时推送 / Workflow Core 审批流 / Polly 熔断
- **高并发处理**：
  - **限流防刷**：Redis 滑动窗口限流，按手机号+IP+设备指纹多维度分桶，峰值期登录稳定率 99%+
  - **请求削峰**：视频转码走 Redis 队列异步处理，主流程不阻塞，等待时长下降 40%
  - **多节点消息同步**：通过 Redis Pub/Sub 跨节点路由，用户无论连接哪个节点都能收到实时状态推送
- **接口幂等性**：考试交卷采用 Redis SETNX 检查考生 ID+试卷 ID 唯一标识，防止重复提交
- **分布式 ID**：雪花算法生成考试订单号，支持多节点并行生成且全局唯一
- **弱网上传**：七牛云分片直传+断点续传，文件上传成功率提升 30%
- **通用审批流**：支持按部门/角色/身份配置审批人，层级自动流转

#### 多租户电子发票网关（.NET 10 + MongoDB + 工厂模式）

> 为多地区、多渠道提供统一开票服务

- **核心技术**：工厂模式 / MongoDB TTL 缓存 / .NET Framework 4 双框架支持 / Redis 分布式锁
- **高并发与一致性处理**：
  - **缓存策略**：MongoDB TTL 索引缓存第三方开票结果，第三方 API 调用次数下降 50%
  - **缓存一致性**：延迟双删策略保证缓存与数据库一致，开票结果先删缓存再更新 DB，延迟 200ms 二次删除
  - **接口幂等性**：开票请求带业务流水号，用 Redis SETNX 检查是否已处理，防止重复开票
  - **开票防锁**：多渠道开票用 Redis 分布式锁串行化，避免并发导致的状态异常
- **可扩展设计**：新增开票渠道只需注入实现类，零改动核心代码（工厂模式）
- **成果**：第三方 API 调用次数下降约50%

#### HDIS 血液透析信息系统（SaaS）（ASP.NET Core + Admin.NET + Vue3）

> 医疗行业 SaaS 系统，支持多租户隔离与 RBAC 权限管理
