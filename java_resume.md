## 项目经历

### 核心项目

#### 河南产权交易中心分布式微服务系统（Spring Cloud Alibaba）

> 基于 Spring Cloud Alibaba 的多节点分布式微服务系统，Nacos 注册/配置中心，涵盖专家培训考试、电子发票网关、金融保函、数据可视化四条业务线，各子服务独立部署并通过 API 网关统一认证与路由。

**API 网关服务（Spring Cloud Gateway + JWT 统一认证）**

系统统一入口，所有客户端请求经网关认证后转发至各子服务。
- JWT 认证链路：用户登录 → 网关校验账号密码 → 签发 Access Token + Refresh Token → GlobalFilter 解析 Token → 将用户上下文写入请求头，Feign 声明式调用透传至子服务
- Token 管理：Redis 维护黑名单支持主动踢人，Refresh Token 续签；统一处理跨域、限流与请求日志

**专家培训与考试服务（Spring Boot + Redis + FFmpeg + WebSocket）**

支持考试高峰期集中登录、视频学习和交卷提交的核心业务服务。
- **限流防刷**：Redis 滑动窗口限流，按手机号+IP+设备指纹多维度分桶，峰值期登录稳定率 99%+
- **视频转码**：七牛云 OSS 分片直传 + 断点续传，Webhook 回调触发 Redis 队列，FFmpeg 异步转码多分辨率（720p/1080p），主流程不阻塞
- **跨节点通信**：Redis Pub/Sub 构建跨节点消息路由总线，多实例 WebSocket 实时推送转码进度与考试状态
- **接口幂等**：Redis SETNX 检查考生 ID + 试卷 ID 防重复交卷；基于 Snowflake 雪花算法生成分布式考试订单号
- **通用审批流**：集成 Flowable 工作流引擎，建模审批节点、条件过滤器与层级流转；最多五级审核，每级可选当前部门负责人、上级部门负责人或指定某个人，各业务单据独立配置审批流程

**多租户电子发票网关服务（Spring Boot + MongoTemplate + 工厂模式）**

聚合百望、金蝶等第三方税务厂商的统一开票服务。
- **可扩展设计**：工厂模式标准化异构接口，新增渠道注入实现类即可，零改动核心流程
- **缓存降本**：MongoTemplate + Query/Criteria 按租户+请求哈希缓存第三方响应，`Index().expire()` 设置 7 天 TTL 自动清理；Redis SETNX 业务流水号幂等校验防重复开票
- **并发安全**：Redis 分布式锁串行化多渠道开票请求
- **安全传输**：SM4 加密保障报文机密性，摘要算法校验防篡改

**金融保函核心业务服务（Spring Boot + 工厂/策略模式）**

面向保险机构的保函开具与管理，处理高敏感度金融承保数据。
- 工厂+策略模式构建第三方财险对接网关，抽象 `IGuaranteeService` 契约解耦厂商 SDK，新增机构零侵入
- SM4 加密 + 摘要算法保障大容量业务报文机密性与防篡改

**GoView 数据可视化服务（Spring Boot + 多数据库适配）**

统一查询中间层，前端传入结构化查询参数（表名、列名、比较运算符），后端动态生成 MySQL、SQL Server、MongoDB 等不同数据库方言的查询语句，屏蔽底层差异

**共享基础设施**
- Nacos 服务注册发现 + 配置中心动态管理各服务配置
- Redis 缓存 / Sentinel 流量控制 / Feign 声明式服务间调用
- Sensor + 统一 ELK 日志采集与慢接口定位

#### HDIS 血液透析信息系统（Spring Cloud Alibaba 微服务架构）

> 面向血液透析中心的分布式医疗 SaaS 平台，基于 Spring Cloud Alibaba 微服务架构，Nacos 作为注册与配置中心，覆盖接诊、透析排班、处方开立到结算全流程，按分院/租户实现数据隔离。核心拆分为 API 网关服务、HDIS 核心服务、ETL 数据集成服务三个独立部署的子服务，统一通过 Nacos 注册发现。

**API 网关服务（Spring Cloud Gateway + JWT 认证）**

系统统一入口，所有客户端请求经网关认证后转发至核心服务。
- **JWT 认证链路**：用户登录 → 网关校验账号密码 → 签发 Access Token（30min）+ Refresh Token（7d）→ Redis 存储 Refresh Token 并维护黑名单 → 后续请求携带 Access Token → Gateway GlobalFilter 解析校验 → 将 userId + tenantId 写入请求头，Feign 声明式调用透传至核心服务
- **Token 管理**：支持主动踢人下线（Token 加入 Redis 黑名单）、Refresh Token 续签；对外屏蔽内部服务拓扑，统一处理跨域、IP 黑名单与请求日志

**HDIS 核心服务（Spring Boot + MyBatis-Plus）**

平台业务主体，承载患者管理、透析排班、处方开立、费用结算、CNRDS 国家平台上报等核心模块。
- **RBAC 权限控制**：用户 → 角色 → 菜单/按钮五表模型，后端通过 `@PreAuthorize` 方法级鉴权拦截越权请求，前端路由按权限表动态生成菜单
- **多租户数据隔离**：MyBatis-Plus 拦截器从网关透传的请求头中解析租户 ID，自动注入 SQL 条件实现按分院/租户逻辑隔离，业务代码零侵入
- **查询性能优化**：针对透析记录/费用报表等高频查询，结合执行计划补充复合索引与覆盖索引，减少回表和排序开销，典型查询耗时由约 1s 降至 200ms 左右
- **缓存策略**：Spring Cache + Redis 缓存预计算统计指标，热点数据响应提升明显；多租户缓存隔离，按租户 Key 拆分缓存空间
- **药品/耗材库存管理**：库存按药品编码 + 库位 + 批号 + 效期多维度管理，支持入库、出库、移库、盘点、冻结/解冻等库内操作，乐观锁机制保障并发场景下库存数据一致性
- **CNRDS 国家平台上报**：梳理患者基本信息、诊断、治疗方案、用药、实验室检查与透析治疗记录等结构化数据；实现字段映射、必填/范围/编码规则校验、异常数据拦截、批量补报、失败重试与上报日志追踪，保障病例信息按时、准确、完整报送
- **中间件使用**：Elasticsearch 实现病历全文检索与疾病统计，RabbitMQ 实现处方开立与配药的异步流转

**ETL 数据集成服务（Spring Boot + XXL-Job）**

独立部署的数据集成服务，对接 HIS 等外部医院信息系统。
- 处理单位统一、编码映射、缺失值校验等数据清洗，将数据错误率降低约 15%
- XXL-Job 分片调度处理批数据，失败重试 + 异常告警保障关键医疗数据完整传输，Logback 详细记录处理追踪

**共享基础设施与部署**
- Nacos 服务注册发现 + 配置中心动态管理各服务配置
- 自研通用监控基础设施——Spring AOP 切面记录 API 请求耗时、MyBatis 插件拦截器记录 SQL 执行耗时，统一写入 Logback → ELK 用于慢接口定位
- RabbitMQ 消息持久化 + 消费确认保障处方异步流转可靠性
- Docker + K8s 容器化部署，Harbor 镜像仓库 + Helm 脚本编排，参与澳门分院跨网环境交付
